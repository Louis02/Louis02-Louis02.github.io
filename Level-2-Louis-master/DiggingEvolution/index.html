Digging Evolution

by: Louis Derrien

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

The game dig dug had always intrigued me and I had relished playing it online on numerous occurrences. I had been aspiring to produce a game like that but more manageable to play. That's when this game had popped into my head. When I first imagined it, it was a vague sketch but the more I advanced in the code the more clear the game came to me. At the completion of it, I had created a game I am proud to display to everyone.

2. What I Learned

I regularly got simple adjustments that could be supplemented through several lines of code. An example of this is to inform the person how much visibility he or she had. However, when one person was testing it out they overlooked entering their username. This caused the game to malfunction when they ran out of lives. I made it so it reminded the player to enter their username if they hadn't previously. If they didn't input their username, then it wouldn't allow them to play.

3. Architecture of My Game

I have 5 distinct classes that each have a diverse purpose. The three central classes are the Runner, Game Panel, and Object Class. The Runner lets the program run and establishes measurements for the game. The following class is called the Game Panel Class. This is my largest class because it holds all the code for the game itself. It arranges each individual game state and makes the main character, Jeff move. It holds all the code for performing the game. The succeeding class is the Object Class and it accommodates all the graphics. It possesses graphics for Jeff, the sky, the dirt, and more. It establishes what they look like and what they're described as. The next class sets up the algorithm for the leaderboard at the end of the game. It is called the Leaderboard Class. It does the math and embeds the scores into the leaderboard. The final class is the Object Manager Class. This class creates the array list for the object and attaches them to the Object Class. The Runner is the main class and all the code is inputted toward it and the Runner runs it to create the game. The Object supplies the graphics to the Game Panel Class and the Object Manager sets up the objects for the Object class. The Leaderboard Class gives the code toward the Game Panel.

4. Technical Implementation

One of my favorite parts of the game is how I was able to make the Abrikoos (The object that Jeff is supposed to eat) fade away as Jeff moved away for it. When I first imagined doing this I didn't have graphics yet so I had changed the color to make it darker and darker until it matched the background and therefore invisible. Then when I added graphics I aspired to have the Abrikoos have the impression of fading away. I first began by forming an array list in the Object Class with the different opaqueness for the Abrikoos. Then I made it invisible if Jeff was too far away to see. I did this code in the Game Panel. I then created an algorithm to decrease the visibility of the Abrikoos every time Jeff got closer. Finally, whenever I moved up a level I would decrease the overall visibility by 1 block.

5. Most Challenging Part

One of the most challenging elements of this game was substituting where Jeff was after he relocated. To do this we had to trace his location on the grid and replace wherever he was latest. It was troublesome to determine where to embed this code but following some trial and error, I put it after every time he moved in any direction.

6. Future Fixes and Updates

I would like to make the clouds move randomly whenever someone moved Jeff. I also would like to understand how to be able to remove text when registering a username.
